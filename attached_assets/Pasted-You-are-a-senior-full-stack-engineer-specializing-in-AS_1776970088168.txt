You are a senior full-stack engineer specializing in ASP.NET and Angular.

Build a production-ready full-stack web application for a "Project Evaluation Platform".

The platform allows users to submit projects, vote on them, and includes authentication, roles, payments, and an admin dashboard.

---

## 🧱 Tech Stack (STRICT REQUIREMENT)

Backend:
- ASP.NET Core Web API (.NET 8)
- Entity Framework Core
- SQL Server
- JWT Authentication

Frontend:
- Angular (latest stable version, NOT AngularJS legacy)
- TypeScript
- Angular Material or TailwindCSS

---

## 🎯 Core Features

### 1. Authentication & Roles (CRITICAL)
- Register / Login system
- JWT-based authentication
- Roles:
  - Admin
  - User
- Role-based authorization

Users can:
- Register
- Login
- Submit projects
- Vote

Admins can:
- Manage users
- Approve/reject projects
- View analytics

---

### 2. Project System

Each project has:
- Title
- Description
- Category
- Image (optional)
- Owner (User)
- Status (Pending / Approved / Rejected)
- CreatedAt

Users can:
- Create project
- View projects
- Only approved projects are public

---

### 3. Voting System

- Each user can vote ONCE per project
- Store votes in DB
- Show total votes per project
- Prevent duplicate voting

---

### 4. Payment Integration

- Use Stripe (test mode)
- Users can:
  - Pay to "feature" their project
- Add:
  - Stripe checkout session
  - Webhook handling

Feature logic:
- Featured projects appear first

---

### 5. Admin Dashboard

Admin panel must include:

- List all users
- List all projects
- Approve / reject projects
- View payments
- View total votes

## 🔌 API Endpoints

Auth:
- POST /api/auth/register
- POST /api/auth/login

Projects:
- GET /api/projects
- POST /api/projects
- PUT /api/projects/{id}/approve (Admin)
- PUT /api/projects/{id}/reject (Admin)

Voting:
- POST /api/projects/{id}/vote

Payments:
- POST /api/payments/create-checkout-session
- POST /api/payments/webhook

Admin:
- GET /api/admin/users
- GET /api/admin/projects
- GET /api/admin/analytics
